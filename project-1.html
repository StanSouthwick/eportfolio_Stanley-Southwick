Project 01

Network Anomaly Detection

An end-to-end production-style ML pipeline that detects and categorises malicious network traffic using a two-tiered ensemble model.

Overview

This is an end-to-end production-style machine learning pipeline designed to detect and classify malicious network traffic. The system is built around a two-tiered ensemble model architecture: the first tier flags whether a given network flow is anomalous, and the second tier categorises the nature of the attack (e.g. DDoS, port scan, brute force).

The pipeline covers the full ML lifecycle from raw packet data through feature engineering, model training, evaluation, and a production-ready inference interface.

Architecture

  • Tier 1 — Anomaly Detection: Ensemble classifier distinguishes normal traffic from anomalous flows using binary classification.
  • Tier 2 — Attack Categorisation: Flagged traffic is passed to a multi-class ensemble model that identifies the specific attack category.
  • Production pipeline: End-to-end design with preprocessing, feature engineering, model serialisation, and an inference layer.

Key Features

  • Two-tiered ensemble approach separates detection from classification for improved accuracy and interpretability.
  • Production-style codebase with modular components for preprocessing, training, and inference.
